Review of the “BSD style” license

  • Example: The Apache Group www.apache.org/LICENSE.txt

  • Allows recipient to do anything with the code

    • recipient obligations limited to including notice, copyright and disclaimer
    • allows “privatization” of code -- recipient may modify the code and treat the modified version as proprietary
    • recipients decide to contribute to the development community for engineering reasons, not due to license requirements
