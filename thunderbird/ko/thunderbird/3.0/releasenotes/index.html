Thunderbird 3.0 새로운 기능

출시일
2009/12/08
Thunderbird 3 의 기능
자세한 정보는 Thunderbird 3 기능 개요를 참고하시기 바랍니다.

Thunderbird 3은 Gecko 1.9.1.5 플랫폼을 기반으로 하고 있어 성능 및 안정성, 웹 호환성 향상 및 소스 코드의 간소화와 지속 가능성을 제공하는 대규모 재 설계를 포함하고 있습니다.

주요 기능은 아래와 같습니다.

검색 기능

  • 고성능 추출 도구를 통한 검색

    검색 결과에 향상된 추출 도구가 탑재되었습니다. 발신인, 태그, 첨부 파일, 참조자, 폴더, 메일링 리스트 등에 의해서 검색 결과를 향상 시킬 수 있습니다.

  • 자동 완성 기능을 통한 검색

    검색창에 문자를 입력하면 주소창을 바탕으로 한 자동 완성 기능을 제공합니다. 전체 내용 혹은 발신인 등 메일 일부를 대상으로 하는지를 선택하는 것도 가능합니다.

사용자 체험 향상

  • 새로운 메일 계정 설정 마법사

    새로운 메일 계정 설정 마법사가 주요한 메일 서비스 제공자의 서버 설정이 등록된 데이타베이스를 참조하므로 새로운 메일 계정을 설정할 때 이름, 메일 주소 패스워드를 입력하는 것만으로 끝납니다.

  • 메일 도구 모음 재설계

    메일 도구 모임이 재설계되어 새로운 검색 도구에 탑재되었습니다. 답장 전송, 삭제, 스팸 메일 등의 버튼은 메세지의 헤더 부분에 이동되었습니다.도구 모음을 맞춤화 하여 다양한 버튼을 추가하는 것도 가능합니다.

  • 메세지 탭 표시

    메세지를 더블 클릭 하거나 선택한 상태로 Enter 키를 누르면 그 메세지가 새로운 탭으로 열립니다. 메세지나 폴더를 백그라운드 탭으로 열 수 있습니다. Thunderbird를 종료할 때 열려 있는 탭 상태가 기억되어 다음 번 재시작 시 복원됩니다. 탭 바의 오른쪽에 열려 있는 탭의 제목을 볼 수 있는 목록이 있습니다.

  • 스마트 폴더

    복수 계정에서 수신 항목 보이기 등 특별한 폴더를 정리해 표시하는 스마트 폴더 모드가 추가되었습니다. 스마트 폴더는 초기 설정으로 제공합니다.

  • 메세지 요약 미리 보기

    복수 메세지를 선택하면 메세지 요약을 볼 수 있습니다.

  • 컬럼 헤더

    컬럼 헤더 표시 항목으로 바꾸어 순서를 폴더 마다 설정할 수 있게 되었습니다.

  • 메세지 저장(archive)

    수신 트레이나 그 외의 폴더로부터 새로운 저장(archive) 폴더 시스템에 메세지를 이동해 보관할 수 있게 되었습니다.

  • 작업 관리자

    작업 관리자는 메일 서비스 제공자와 송수신 로그를 모두 한곳에 정리해 기록합니다.

  • 새로운 부가 기능 관리자

    새로운 부가 기능 관리자 ([도구] > [부가 기능])에서 확장 기능, 테마, 플러그 인 등의 기능를 직접 검색하고 설치 할 수 있게 되었습니다. 덧붙여 부가 기능은 각 개발자가 Thunderbird 3를 지원하여야 이용할 수 있으므로 출시 당시에 대응하지 못하는 부가 기능도 있습니다.

  • 향상된 주소록

    발신인 등이 주소록에 등록되어 있으면 새로운 별 표시가 보이며, 이를 클릭하면 연락처 상세 항목을 편집할 수 있습니다. 주소록에 등록하지 않은 경우는 아이콘을 클릭하는 것만으로 연락처로 추가할 수 있습니다. 생일 및 사진도 추가할 수 있게 되었습니다.

  • Gmail 통합 기능

    Gmail 언어가 영어 이외에 설정되며 보낸 편지함이나 지운 편지함 등 폴더가 올바르게 인식되지 않았던 문제가 수정되었습니다. Thunderbird 3에서는 모든 편지함 폴더가 저장 (archive) 폴더로서 사용됩니다.

  • Windows Vista 및 Windows 7에서 윈도우 의 검색 결과에 Thunderbird 3 에서 저장하고 있는 메세지가 표시되게 되었습니다. Thunderbird 는 처음 시작 시 윈도우에 메일 목록을 설치 할지를 묻습니다. 이후 윈도우 검색 결과로부터 메일이나 뉴스의 메세지를 선택할 수 있게 됩니다.
  • 맥킨토시에서는 Thunderbird 3를 Spotlight와 통합되는 것과 동시에 Apple Mail에서의 환경 설정을 통해 새 메세지의 통지가 가능합니다.

성능 향상

  • IMAP 폴더 동기화

    오프 라인시 메일을 읽을 수 있도록 Thunderbird내 IMAP 메세지를 백그라운드로 받아와 동기화 할 수 있습니다. 이 기능은 폴더의 설정을 통해 폴더 마다 할 수 있으며 계정 설정의 동기화 및 디스크 영역에서 계정 내 모든 폴더에 대해서 설정하는 것도 가능합니다.

시스템 요구 사항

설치 전에 사용자 컴퓨터가 시스템 요구 사항을 충족하는지 살펴봐 주십시오.

Thunderbird 3은 2000년도 이전의 윈도우 버전 (Windows 95, 98, ME,NT) 및 10.4 Tiger 이전의 Mac OS X에서는 제공하지 않습니다. 본 운영 체제에 대해서는 Thunderbird 2 등 이전 버전을 이용해 주십시오.

다운로드

Mozilla는 Thunderbird 3을 윈도우, 리눅스 및 맥 OS X와 다양한 언어 버전을 제공 합니다. 지급 바로 Thunderbird 3 최신 버전을 다운로드 하세요. 다른 종류의 운영 체제나 언어 버전은 Mozilla.org에서 공식적으로 제공하지 않습니다. 아래 기타 버전을 부분을 참고하세요.

설치g하기

Thunderbird를 설치하면 Mac OS X, Linux, Windows 에서 이미 설치한 Thunderbird에 덮어 쓰는 것에 주의해 주십시오. 하지만 북마크나 방문 기록 등에는 일체 영향 없지만, 확장 기능이나 그 밖의 부가 기능 일부는 개발자가 업데이트를 제공할때 까지 동작하지 않을 가능성이 있습니다. 향후에 원하시면 하위 버전을 다시 재설치 하실 수 있습니다. .

이미 Thunderbird를 사용하고 있어서 Thunderbird 3를 테스트하는 경우 Thunderbird 3를 다른 폴더에 설치 해 (사용자 정의 설치)를 하고 프로필을 백업 하는 것을 추천 합니다.

프로그램 제거

윈도우 사용자는 시작 메뉴에서 제어판을 찾아 Mozillla Thunderbird를 제거하실 수 있습니다. 맥 OS 에서는 Thunderbird 애플리케이션을 휴지통에 버리고 리눅스에서는 thunderbird 폴더를 삭제하면 됩니다.

삭제하더라도 북마크나 방문 기록, 확장 기능 및 부가 기능은 삭제되지 않으며, 프로필 폴더라는 곳에 저장됩니다. 각 운영 체제별로 프로필 폴더의 위치는 아래와 같습니다.

Windows VistaUsers¥<사용자명>¥AppData¥Roaming¥Thunderbird
Windows 2000 XPDocuments and Settings¥<사용자명>¥Application Data¥Thunderbird
Windows NTWINNT¥Profiles¥<사용자명>¥Application Data¥Thunderbird
Windows 98 MEWindows¥Application Data¥Thunderbird
Mac OS X~/Library/Thunderbird
Linux 및 Unix 시스템~/.thunderbird

Thunderbird 3을 삭제한 후에 다시 Thunderbird를 설치 했을 경우 그 버전에 관계없이 계속해서 프로필 폴더에 있는 데이터를 사용 합니다.

확장 기능 및 테마

Thunderbird 이전 버전에 설치된 확장 기능 중에는 Thunderbird 3에서 이용하기 위해서 업데이트가 필요한 것이 있습니다. 문제가 있는 경우 사용하고 있는 확장 기능의 관리자에게 연락해 주십시오.

Thunderbid를 설치하면 모든 확장 기능과 테마는 호환성이 있는 새로운 버전이 제공 될 때 가지 이용할 수 없게 됩니다.

알려진 문제점

Thunderbird 3에서 알려진 문제점 내역을 알려 드립니다. 이 버그들은 향후 보안 및 성능 업그레이드 버전을 통해 계속 수정할 예정입니다.

모든 시스템
  • 과거에 기본 암호 기능을 사용했던 적이 있는 경우 Thunderbird 의 첫번째 실행 시에 기본 암호를 입력을 요구 할 수 있습니다. (Bug 338549)
  • Thunderbird 3의 암호는 새로운 곳에 저장 합니다. 이전 버젼으로 돌아오거나 Thunderbird 3의 이전 버전을 교대로 사용했을 경우 암호를 재사용할 수 없습니다.
  • 영어 외 언어에서는 메세지의 검색은 대문자와 소문자가 구별됩니다. (Bug 525537)
  • SSL 혹은 TLS를 사용해 서버로부터 제공된 인증서가 서명 기한 마감 혹은 서버의 도메인과 일치하지 않는 경우 Thunderbird에서는 인증서를 예외로서 받아 들일지를 묻는 대화 상자가 표시됩니다. 인증서를 믿는 경우에만 예외 사항으로 추가하시기 바랍니다.
  • 확장 칼럼은 Beta 1에서 삭제되었습니다. 이 기능을 제공하는 부가 기능이 공개되어 있습니다.
  • 콘파뷸레이터 Beta 3은 삭제되었습니다 (Bug 480623).자세한 것은 이 블로그 기사를 봐 주세요.
  • Thunderbird 3에서 업데이트한 사용자로부터 송신 (SMTP) 서버에 로그인할 수 없는 경우가 있습니다. 이는 보호된 인증 설정이 유효하게 되어 있어 SMTP 서버가 이를 지원하지 않기 때문입니다. [도구] 메뉴에서 [계정 설정] > [송신 (SMTP) 서버]를 선택해 서버 설정을 확인해 문제가 있는 서버의 설정을 수정해 주시기 바랍니다. 「보호된인증을 사용」에 체크가 들어가 있는 경우는 그 체크를 제외하고 나서 재차 송수신 테스트를 해 주시기 바랍니다.(Bug 522633)
  • Thunderbird는 HTTPS 프로토콜을 사용한 안전한 서버로부터 제공하는 서명한 부가 기능만을 받아 들입니다. 조건에 맞지 않는 경우 설치할 수 없습니다.
  • 길게 표시하는 레이아웃으로 RSS 피드의 컨텐츠를 표시할 수 없는 경우 Lightning 캘린더 부가 기능를 설치 하고 있는 경우 기본 표시로 바꾸고 나서 Thunderbird를 재시작 할 필요가 있습니다.(Bug 531397)
  • 기본 텍스트로 받은 일부 메세지에 대해서 메세지나 버튼의 일부가 메세지 표시 밖에 있는 경우가 있습니다. 메세지를 새로운 탭으로 열리는지 윈도우 크기를 바꾸는지 레이아웃을 기본 표시 혹인 길게 표시를 바꾸어 메세지를 표시할 수 있습니다.(Bug 530047)
  • 오프라인 작업 중 메세지를 편집해 나중에 송신을 선택하면 인터넷에 자동적으로 재접속해도 메세지가 송신되지 않는 경우가 있습니다. 그 경우 작성 중 폴더로부터 수작업으로 메세지를 재발송신 할 필요가 있습니다.(Bug 531179)
  • 특정 상황에서 파일 메뉴로 이용할 수 있는 템플릿 기능이 작동하지 않는 경우가 있습니다. 메일의 편집 화면에서 템플릿 기능을 선택하거나 메세지를 템플릿 폴더에 드래그앤 드롭 하는 것으로 이 기능을 이용하는 것은 가능합니다.(Bug 532050)
Microsoft Windows
  • 전체 시작 바로 가기 기능이 Ctrl+Shift+C에서 Shift+C로 변경되었습니다.

문제 해결 방법

  • 확장 기능에 따라서는 안정성이나 성능 저하 저하하는 등의 문제를 일으키는 일이 있습니다. Thunderbird 의 일부 기능을 사용할 수 없거나 윈도우 표시가 이상한 성능 저하 등의 문제를 만났을 때 확장 기능이나 테마가 원인일 가능성이 있습니다. 이 때 Thunderbird를 안전 모드로 재시작 해 주십시오.

    윈도우 시작 메뉴내에 「Mozilla Thunderbird (안전모드)」라는 바로 가기를 사용하거나 thunderbird.exe -safe-mode를 통해 시작할 수 있습니다. Linux에서는 ./thunderbird -safe-mode로 Mac OS X 터미널을 열어 실행해 주면 됩니다.

    cd /applications/Thunderbird.app/Contents/MacOS/
    ./thunderbird-bin -safe-mode

    안전 모드로 시작하면 확장 기능은 모두 무효화되고 기본 테마가 사용됩니다. 문제의 원인이 된 확장 기능이나 테마를 무효화하고 나서 이전과 같이 시작해 주시기 바랍니다.

  • 메일, 피드, 뉴스 그룹, 메세지 등에 대해 이상한 문제를 만난 경우 새로운 사용자 프로필을 작성해 문제가 재현되는지 확인해 주세요. 새로운 프로필을 작성하려면 명령어 인수 -P를 붙여 Thunderbird를 시작해 [새 프로필 작성] 버튼을 클릭해 주세요. 설정 파일 (메세지 피드 등)를 바꿀 때 마다 문제가 발생하는지를 확인해 주세요. 특정 프로필 데이터 파일이 문제의 원인이라고 생각 되면 버그로서 보고 하고 문제 파일을 첨부해 주세요. 이 때 가능하면 반드시 개인정보를 삭제해 주시기 바랍니다.

자주 묻는 질문

  1. 개발 프로젝트에 참여하려면 어떻게 하나요?

    Thunderbird를 더욱 좋은 제품으로 만들기 위해 각종 의견을 모으고 있으며 개발자 여러분이나 테스트 커뮤니티의 지원을 필요로 하고 있습니다. Bugzilla에 버그를 보고하기 전에 이 제품 보고서 및 버그 등록 방법을 읽어 주시고 여러분의 의견을 보내 주십시오.

  2. 확장 기능이나 테마(애드온(ad-on))은 어디에서 받습니까?

    확장 기능 및 테마 사이트에서 다운로드 가능합니다. 특히 Thunderbird 부가 기능 페이지를 방문하십시오.

  3. 누가 Thunderbird 3을 개발하고 있습니까?

    매우 많은 사람들이 관련되어 있고 Thunderbird 3에 공헌하고 있는 사람들의 목록을 보기 위해서는 도움말 메뉴의 Mozilla Thunderbird 정보의 공헌자 항목을 보시기 바랍니다.

  4. Thunderbird 소스 코드는 어디에 있습니까?

    Thunderbird의 소스 코드는 압축 파일 다운로드를 이용하거나 최신 개발 코드의 경우 Mercurial 이라는 소스 코드 시스템에서 얻을 수 있습니다. Thunderbird 소스 코드는 comm-central의 "mail", "toolkit", "chrome" 디렉토리에 있습니다. 직접 만들어 보시려면 빌드 방법을 참고하세요.
