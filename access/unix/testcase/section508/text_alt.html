Test Case Description for  Text_only pages

Purpose: If accessibility cannot be accomplished in any other way, provide a text-only page with equivalent information or functionality.
               Update the content of the text-only page whenever the primary page changes.



Initial  Conditions:
  1. Have Screen Reader program installed.

Steps/Description:
  1. Compare the primary Web page with the text-only Web page every time an update is done to the primary Web page.


 
Expected Results:
  1. The primary Web page and the text-only page will have equivalent information and functionality.
