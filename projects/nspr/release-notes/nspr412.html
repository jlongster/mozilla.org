1. Introduction

The NSPR 4.1.2 release is now available at /share/builds/components/nspr20/v4.1.2. This release is a patch release for the NSPR 4.1 release.

2. Changes in this release

2.1 Bug fixes

The following bugs are fixed in this release.

Bugzilla #86941. Closing a TCP socket may fail with EINVAL.
Bugzilla #83845. IPv6 support returns wrong data for getpeername
Bugzilla #83930. A typo in PR_LockFile in pr/src/io/prfile.c
Bugzilla #86880. NSPR 3.5.1 and 4.1 do not recognize ECONNABORTED

 3. Reporting bugs

Bugs discovered should be reported by filing bug reports.  If you can describe a bug without giving any proprietary information, you should file the bug report in mozilla.org's bug database Bugzilla.  You will need a Bugzilla account, which can be obtained easily from the Bugzilla main page.  If proprietary information cannot be avoided in a bug description, you can file the bug report in Bugsplat.

The following information is required in a bug report

  • platform (Solaris 2.6, HP-UX 11.0, etc.)
  • number of cpus in the system
  • a stack trace, in the event of a crash
  • reproducibility of the problem
  • location of core dump, if available, along with those for the libraries and executables

 4. Building from source

This release is built from the source at the CVS repository cvs.mozilla.org:/cvsroot with the CVS tag NSPRPUB_RELEASE_4_1_2. Check out the source from CVS by

    cvs co -r NSPRPUB_RELEASE_4_1_2 mozilla/nsprpub

To build, do
    cd mozilla/nsprpub
    gmake export
