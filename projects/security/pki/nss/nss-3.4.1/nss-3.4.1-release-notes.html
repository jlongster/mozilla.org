Introduction

Network Security Services (NSS) 3.4.1 is a patch release for NSS 3.4. Please refer to the NSS 3.4 Release Notes for the common information. The bug fixes in NSS 3.4.1 are described in the "Bugs Fixed" section below.

Distribution Information

The CVS tag for the NSS 3.4.1 release is NSS_3_4_1_RTM.

NSS 3.4.1 source and binary distributions are also available on ftp.mozilla.org for anonymous ftp download:

  • Source tarballs: ftp://ftp.mozilla.org/pub/mozilla.org/security/nss/releases/NSS_3_4_1_RTM/src/.
  • Binary distributions: ftp://ftp.mozilla.org/pub/mozilla.org/security/nss/releases/NSS_3_4_1_RTM/. Both debug and optimized builds are provided. Go to the subdirectory for your platform, DBG (debug) or OPT (optimized), to get the tar.gz or zip file. The tar.gz or zip file expands to an nss-3.4.1 directory containing three subdirectories:
    • include - NSS header files
    • lib - NSS shared libraries
    • bin - NSS Tools and test programs
You also need to download the NSPR 4.1.2 binary distributions to get the NSPR 4.1.2 header files and shared libraries, which NSS 3.4.1 requires. NSPR 4.1.2 binary distributions are in ftp://ftp.mozilla.org/pub/mozilla.org/nspr/releases/v4.1.2/.

Bugs Fixed

The following bugs have been fixed in NSS 3.4.1.
  • Bug 135721: nss_ckmdSessionObject_GetAttribute should return NSSCKFWItem.
  • Bug 135728: Fixes for three benign compiler warnings.
  • Bug 136253: Rainbow accelerator certificates are not visible.
  • Bug 138084: NSC_Initialize leaves a slot list even if it fails.
  • Bug 138354: trust bits are zero for certs on nCipher token with NSS 3.4.

Documentation

For a list of the primary NSS documentation pages on mozilla.org, see NSS Documentation. New and revised documents available since the release of NSS 3.3.1 include the following:
  • Build Instructions for NSS 3.4.1 describe the new NSS 3.4.1 CVS tag and other minor changes since NSS 3.3.2.

Compatibility

NSS 3.4.1 shared libraries are backward compatible with NSS 3.4, NSS 3.3.x, and NSS 3.2.x shared libraries. A program linked with NSS 3.4, NSS 3.3.x, or 3.2.x shared libraries will work with NSS 3.4.1 shared libraries without recompiling or relinking.  Furthermore, applications that restrict their use of NSS APIs to the functions listed in NSS 3.4 Public Functions will remain compatible with future versions of the NSS shared libraries.

Feedback

Bugs discovered should be reported by filing a bug report with bugzilla (product NSS).

You can also give feedback directly to the developers on the IRC channel #mozcrypto on the server irc.mozilla.org.
