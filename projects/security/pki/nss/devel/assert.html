This is a policy statement about two closely related topics: assertions and invalid argument (such as null pointer) checking.

Our policy on asserts had the following components:

1. For every assertion, there must be code that follows it that avoids a crash for the asserted condition in the "optimized" builds.

2. Assertions should catch programming errors that occur inside NSS itself, but not in external callers.  NSS should return error codes to external callers, not crash when they err.   But NSS should assert the correctness of its own internal callers.

3. Assertions should not be used to catch erroneous data being passed in.  That is, erroneous data should cause error codes, not crashes, even in debug code. This only applies to functions that return an indication of success. This applies to external functions, not internal ones.
