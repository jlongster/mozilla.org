He needs to follow these instructions.
  • Go to https://despot.mozilla.org/ and log in with his CVS account name (his email address, with @ as opposed %) and CVS password. If he forgets his CVS password, open a bug report against the product mozilla.org for help.
  • In the Despot main menu, click the button "List all the partitions".
  • In the list of partitions, find the partition named "security" and click the "Edit" button next to it. The "security" partition controls the trunk of NSS and JSS.
  • Add your CVS account name (your email address, with @ as opposed to %) to either Owner, Peer, or Member.
  • Click the "Save changes" button.
  • Repeat these steps for the "NSS Stable Release Branch" partition, which controls the current NSS stable release branch.

When checking in code

The CVS commit comment must include the following information.
  • The bug number.
  • A concise description of the checkin. One should be able to understand what the change is without looking up the bug report.
  • The code reviews you received, using the r=reviewer notation.
  • An acknowledgement if the patch was contributed by someone outside the team
  • List of files modified.

Update the Bug

  • Include the CVS output in the bug report. This provides a way to tie a specific version back to the code.
